# Self-Healing Horizon

Self-healing is on the horizon, but it must not arrive before policy hardening
and adapter dry-run enforcement.

Current rule:

```text
No self-healing mutation before dry-run proof.
```

ASF-R can prepare for self-healing by producing wound packages, governance debt,
next admissible actions, replay reports, and policy-bound evidence.

Future self-healing must be constrained by:

- rehydration,
- RCC route,
- active policy,
- capability token,
- authorization receipt,
- adapter dry-run report,
- wound closure evidence,
- ledger record,
- operator-visible UI state.

Self-healing must start as proposal generation, not mutation.
