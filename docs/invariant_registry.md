# Invariant Registry

ASF-R v0.1.1 adds non-negotiable runtime invariants.

- `ASF-INV-001`: Unknown is not pass.
- `ASF-INV-002`: No rehydration, no action.
- `ASF-INV-003`: No RCC route, no mutation.
- `ASF-INV-004`: No policy, no permission.
- `ASF-INV-005`: No wound package, no blocked-state closure.
- `ASF-INV-006`: No ledger record, no durable decision.
- `ASF-INV-007`: No human authorization, no durable authority.
- `ASF-INV-008`: No adapter may self-authorize.
- `ASF-INV-009`: No UI may display pass if decision is blocked.
- `ASF-INV-010`: No README claim may exceed evidence state.

Non-claim lock: invariant checks govern continuation; they do not prove truth.
